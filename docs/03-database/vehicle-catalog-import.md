# Vehicle Catalog Import (CSV)

## Purpose

Load the generated vehicle catalog from `assets/json` into Skaftin database tables in manual, repeatable increments using CSV files.

## Tables Created

Project schema contains these catalog tables:

- `vehicle_makes_catalog`
- `vehicle_models_catalog`
- `vehicle_trims_catalog`
- `vehicle_engines_catalog`
- `vehicle_transmissions_catalog`
- `vehicle_fluids_catalog`

Each table now includes an auto-incrementing `id` column (`BIGINT GENERATED BY DEFAULT AS IDENTITY`) for import/order convenience.
Domain UUID columns (`make_id`, `model_id`, `trim_id`, etc.) remain in place and are still used for cross-table relationships.

Foreign keys:

- `vehicle_models_catalog.make_id -> vehicle_makes_catalog.make_id`
- `vehicle_trims_catalog.model_id -> vehicle_models_catalog.model_id`
- `vehicle_engines_catalog.model_id -> vehicle_models_catalog.model_id`
- `vehicle_transmissions_catalog.model_id -> vehicle_models_catalog.model_id`
- `vehicle_fluids_catalog.model_id -> vehicle_models_catalog.model_id`

## CSV Export

Run:

```bash
npm run export:vehicle-csv
```

Output folder:

- `assets/csv/vehicle-catalog/vehicleMakes.csv`
- `assets/csv/vehicle-catalog/vehicleModels.csv`
- `assets/csv/vehicle-catalog/vehicleTrims.csv`
- `assets/csv/vehicle-catalog/vehicleEngines.csv`
- `assets/csv/vehicle-catalog/vehicleTransmissions.csv`
- `assets/csv/vehicle-catalog/vehicleFluids.csv`

If any source JSON file is missing, the exporter still writes a header-only CSV for that entity.

## Incremental Manual Import Order

Import CSV files in this order to satisfy FK dependencies:

1. `vehicleMakes.csv`
2. `vehicleModels.csv`
3. `vehicleTrims.csv`
4. `vehicleEngines.csv`
5. `vehicleTransmissions.csv`
6. `vehicleFluids.csv`

## Validation Notes

The exporter prints warnings for:

- Missing required mapped fields
- Broken cross-file relations (e.g. trims referencing non-existent models)

Warnings do not stop export so you can still review and manually import valid portions incrementally.
